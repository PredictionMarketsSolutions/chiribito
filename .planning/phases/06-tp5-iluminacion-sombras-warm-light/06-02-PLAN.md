<objective>
Wire the ?light= A/B flag and reshape the key light into a warm gradient (not a cone) behind it.
The shaped path widens the key angle (0.62 to 0.72), raises fill proportionally (0.7 to 0.8)
to maintain the anti-casino floor, and tints the hemisphere ground to dark felt-green (#0d3d24)
for the green-bounce GI effect on object undersides. The base path (?light=base) restores the
prior flat-warm key exactly.

Purpose: The key-to-fill ratio ceiling (KEY_TO_FILL_RATIO_CEILING = 3.5) is the anti-casino
invariant — the key can never pool to a tight cone over dark surroundings. The green-bounce
is delivered via the hemisphereLight ground tint (no new light, no new shadow-caster). This
workstream is separable from the grounding (06-01) — if the shaped key reads as a spotlight,
grounding still ships and this is reverted to base. Non-blocking.

Output: TableLab.tsx with ?light= flag wired, Lights component accepting lightFlag prop,
KEY_TO_FILL_RATIO_CEILING constant, shaped vs base branching for key angle/intensity/fill/hemisphere.
</objective>
